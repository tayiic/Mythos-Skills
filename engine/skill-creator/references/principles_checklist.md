# P1-P8 Checklist

- P1: Is `SKILL.md` compact enough to activate frequently?
- P2: Are details disclosed progressively through local references?
- P3: Can the skill directory be copied alone and still work?
- P4: Are fragile operations constrained?
- P5: Does the description say when to use the skill?
- P6: Are known mistakes and gotchas documented?
- P7: Are repeatable tasks backed by scripts or structured operations?
- P8: Is durable state stored outside the conversation?
