# Creation Workflow

## Phase 1: Trigger

Write the trigger before the body. If the trigger is vague, the skill will not be used reliably.

## Phase 2: Shape

Choose the three-layer layout:

- `SKILL.md` for activation.
- `references/` for detail.
- `operations.json` for repeatable operations.

## Phase 3: Content

Keep the entry point compact. Move examples, long rules, scripts, and background into references.

## Phase 4: Validation

Run the P1-P8 checklist. Fix portability, trigger, line count, and safety issues before publishing.
